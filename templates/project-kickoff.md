# Project Kickoff Request

You are entering an unfamiliar repository.

## Goal
Understand the codebase enough to work safely without over-reading the repo.

## What to learn
- project purpose
- entry points
- main folders
- build / run commands
- test / lint commands
- key dependencies
- obvious risk areas

## Rules
- Read only the minimum useful set of files.
- Prefer actual repo files over assumptions.
- Summarize clearly.
- Flag uncertainty instead of guessing.

## Output format
### Project summary
[2 to 5 lines]

### Key files and folders
- [path] — [why it matters]

### Commands
- build:
- run:
- test:
- lint:

### Risks or unknowns
- [item]

### Suggested next step
- [item]
