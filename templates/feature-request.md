# Feature Request

You are adding a new feature to an existing project.

## Goal
Implement the requested feature in a way that fits the current architecture and style.

## Feature Summary
[Describe the feature clearly.]

## User Outcome
[What should the user be able to do after this is added?]

## Constraints
- Keep existing behavior stable unless change is explicitly required.
- Reuse existing patterns, utilities, and components where possible.
- Keep the implementation easy to review and easy to revert.
- Avoid broad refactors.

## What to do
1. Read only the files relevant to the feature.
2. Explain the likely implementation path briefly.
3. Implement in small, reviewable changes.
4. Add or update validation, error states, empty states, and loading states where relevant.
5. Verify build, lint, tests, and any affected UX flow.

## Output format
- Plan
- Files changed
- What was implemented
- Verification done
- Follow-up improvements
