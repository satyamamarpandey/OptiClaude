# Refactor Request

You are improving code quality without changing intended behavior.

## Goal
Refactor the selected area for readability, maintainability, and lower risk.

## Scope
[Specify the file, module, or workflow.]

## Rules
- No behavior changes unless a bug is discovered and clearly called out.
- No broad repo-wide changes.
- Keep public interfaces stable unless explicitly requested.
- Prefer extracting small reusable functions over large rewrites.

## What to do
1. Identify pain points in the current implementation.
2. Suggest the smallest valuable refactor.
3. Apply the refactor carefully.
4. Verify behavior remains the same.
5. Summarize what improved.

## Output format
- Refactor goal
- Problems found
- Files changed
- What improved
- Verification done
