# Templates Folder

This folder gives you a practical starter set of reusable templates for Claude Code workflows.

## What this is for

Use these templates when you want Claude to work in a more structured way without repeating the same instructions every time.

They are designed to be:

- repo-agnostic
- safe for new users
- easy to customize
- useful across many projects

## Recommended use

Copy the full `templates/` folder into:

```text
%USERPROFILE%\.claude\templates
```

or keep it inside your repository and copy only the files you want.

## Folder layout

- `bugfix-task.md` — for debugging and fixing a problem
- `feature-request.md` — for adding a new feature cleanly
- `refactor-task.md` — for improving code without changing behavior
- `code-review.md` — for review and risk checking
- `test-plan.md` — for verification planning
- `docs-update.md` — for README / docs updates
- `release-checklist.md` — for ship-readiness checks
- `prompt-improve.md` — for rewriting weak prompts into strong ones
- `project-kickoff.md` — for starting work in an unfamiliar repo
- `data/` — example data files you can duplicate and fill in

## How to use

Open a template, replace the placeholders, and paste the content into Claude Code.

You can also keep these as your own saved snippets.

## Notes

These are starter templates, not rigid rules. Edit them to match your workflow.
